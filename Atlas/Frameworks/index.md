# Atlas/Frameworks

Named methods, models, and frameworks: things with a proper name and a defined structure that you apply, not just think about. Examples of the kind: SWOT, the Eisenhower matrix, Zettelkasten itself.

## What belongs here

- One page per framework: what it is, when to use it, how to apply it, where it breaks down.
- Frontmatter: `type: framework`, `up:` (usually the topic it serves), `created-date:`.

## What does not belong here

- A loose idea without a name or defined steps (atomic note in `Atlas/Notes/`).
- Your application of a framework to a specific project (that note lives with the project in `Efforts/` and links here).

## Example

Framework pages appear as you ingest sources that teach methods worth keeping.
